# plex-jarvis — 상세 구현 계획

## 1. 프로젝트 비전

Chrome 확장을 넘어 **로컬에서 작동하는 브라우저 탐지 및 화면 분석 어시스턴트**로 진화. 이미 접속된 URL에 대한 캐싱을 통한 빠른 안내, 개선된 채팅 화면, 누구든 어떤 정보든 이해할 수 있게 해주는 웹 상의 어시스턴트.

## 2. 현재 상태

- Chrome Extension MV3 기반 완성된 프로토타입
- Vanilla JS 단일 파일 구조 (`content.js`에 수집/분석/UI/채팅 집중)
- Gemini API 단일 호출로 최대 50개 요소 분석
- 5초 타이머 기반 자동 활성화
- 기본적인 채팅 기능 (우측 하단 패널)

## 3. 디자인 시스템

**기존 디자인 유지** — Chrome Extension 콘텐츠 스크립트 UI

현재 `content.css` 기반:
- JARVIS 스타일 다크 글래스 오버레이
- 그라데이션 배지, 호버 애니메이션
- 우측 하단 요약/채팅 패널
- 기존 톤과 패턴을 유지하되 UX만 개선

---

## 4. 단계별 구현 계획

### Phase 1 — 구조 개선 + 캐싱 (3주)

**1.1 모듈 분리**
- [ ] `content.js` 해체 → 4개 모듈로 분리:
  - `collector.js` — DOM 요소 수집, 뷰포트 감지
  - `analyzer.js` — Gemini API 호출, 응답 파싱
  - `renderer.js` — 배지, 패널, 채팅 UI 렌더링
  - `cache.js` — URL별 분석 결과 캐시 관리
- [ ] ES Modules 또는 IIFE 패턴으로 모듈 간 통신

**1.2 URL 기반 캐싱**
- [ ] IndexedDB에 URL → 분석 결과 저장
- [ ] 캐시 히트 시 즉시 표시 (API 호출 생략)
- [ ] 캐시 만료 정책 (기본 24시간, 사용자 설정 가능)
- [ ] 페이지 변경 감지 — DOM MutationObserver로 변경분만 증분 분석
- [ ] 캐시 상태 표시 (배지에 "캐시됨" 아이콘)
- [ ] 팝업에서 캐시 관리 (전체 삭제, URL별 삭제)

**1.3 성능 최적화**
- [ ] 뷰포트 기반 우선 분석 — `IntersectionObserver`로 보이는 요소부터 처리
- [ ] 청크 분할 병렬 요청 — 5~10개씩 나누어 동시 API 호출
- [ ] 응답 스트리밍 — Gemini Streaming API 전환
- [ ] 5초 고정 타이머 → `requestIdleCallback` + 페이지 로드 완료 감지로 변경

### Phase 2 — 채팅 UI 개선 (3주)

**2.1 채팅 패널 재설계**
- [ ] 리사이즈 가능한 사이드 패널 (드래그로 크기 조절)
- [ ] 마크다운 렌더링 (코드 블록 하이라이팅 포함)
- [ ] 이미지/링크 프리뷰
- [ ] 대화 이력 로컬 저장 (URL별)
- [ ] 대화 내보내기 (텍스트/마크다운)

**2.2 심화 질의**
- [ ] 맥락 유지 대화 — 페이지 요약 + 이전 대화 맥락 포함
- [ ] 요소 지정 질의 — 특정 요소를 클릭하여 "이것에 대해 더 알려줘"
- [ ] 비교 질의 — "이 약관과 저 약관의 차이는?"
- [ ] 다국어 지원 — 외국어 페이지 자동 감지 → 한국어 요약 옵션

**2.3 에러 핸들링 강화**
- [ ] API 호출 실패 시 재시도 + 사용자 알림
- [ ] 네트워크 오류 시 캐시된 결과로 폴백
- [ ] 토큰 초과 시 자동 텍스트 압축 후 재요청
- [ ] API 키 미설정/만료 시 명확한 안내

### Phase 3 — 로컬 브라우저 탐지 진화 (4주)

**3.1 데스크톱 네이티브 앱 (Electron/Tauri)**
- [ ] 브라우저 독립 네이티브 앱 프로토타입
- [ ] Chrome DevTools Protocol (CDP) 연결 — 브라우저 탭 목록, URL, DOM 접근
- [ ] 멀티 브라우저 지원 — Chrome, Edge, Arc, Brave (Chromium 기반)
- [ ] Firefox: WebDriver BiDi 프로토콜 연동

**3.2 화면 분석**
- [ ] 화면 캡처 → OCR 기반 콘텐츠 추출 (확장 미지원 브라우저 대응)
- [ ] 스크린샷 AI 분석 — 레이아웃, UI 패턴, 접근성 진단
- [ ] 활성 윈도우 감지 (macOS: NSWorkspace, Windows: WinAPI)

**3.3 크로스 브라우저 통합**
- [ ] 여러 브라우저 동시 모니터링
- [ ] 통합 분석 대시보드 — 모든 브라우저의 분석 결과 한 곳에서 관리
- [ ] 브라우저 간 컨텍스트 공유

### Phase 4 — 고급 기능 (3주)

**4.1 페이지 변경 감시**
- [ ] 관심 페이지 등록 → 주기적 변경 감지
- [ ] 변경사항 알림 (가격 변동, 재고 상태, 뉴스 업데이트 등)
- [ ] 변경 이력 타임라인

**4.2 북마크/히스토리 통합**
- [ ] 분석 결과를 북마크처럼 저장
- [ ] 키워드 검색으로 과거 분석 재활용
- [ ] 태그 기반 분류

**4.3 자동 번역 + 요약**
- [ ] 외국어 페이지 자동 감지
- [ ] 전체 번역 또는 핵심 부분만 번역
- [ ] 번역 + 요약 조합 제공

## 5. 기술 스택

### Chrome Extension (Phase 1-2)
| 구분 | 기술 |
|------|------|
| 플랫폼 | Chrome Extension MV3 |
| 언어 | JavaScript (ES Modules) |
| 캐시 | IndexedDB |
| AI | Gemini API (Streaming) |

### 네이티브 앱 (Phase 3-4)
| 구분 | 기술 |
|------|------|
| 프레임워크 | Electron 또는 Tauri 2 |
| 브라우저 연동 | Chrome DevTools Protocol |
| 화면 분석 | Tesseract OCR / Gemini Vision |
